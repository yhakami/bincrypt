# Support

## Getting Help

- Usage questions and setup help: open a GitHub Discussion.
- Reproducible bugs: open a GitHub Issue with environment details and steps.
- Security vulnerabilities: follow the private disclosure process in `SECURITY.md`.

## What to Include in Bug Reports

- BinCrypt version/commit.
- Deployment target (local Docker, Cloud Run, Vercel + backend, Firebase Hosting + backend, etc.).
- Storage backend and rate limiter backend in use.
- Exact request/response or logs (with secrets redacted).
- Minimal reproduction steps.

## Supported Scope

- `main` branch and the latest tagged release are supported.
- Older forks and heavily patched private variants may require self-support.

## Response Expectations

- Best effort for community support.
- No guaranteed SLA for free/open-source support.
